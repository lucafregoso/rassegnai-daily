La notizia arriva lo stesso giorno in cui Apple cerca anche aziende di chip da acquisire per accelerare lo sviluppo di server AI propri, dato che le prestazioni dei suoi server M2 Ultra non soddisfano i requisiti.

Microsoft ha anche cancellato Patch Tuesday per alcuni utenti Dell a causa di shutdown anomali causati dagli update di luglio.

### 4. The Memory Heist: ricercatore tricka Claude facendogli leakare nome, datore di lavoro e domande di sicurezza
Un ricercatore ha dimostrato un attacco contro la memoria persistente di Claude: una singola interazione apparentemente innocua ha portato Claude a inviare il nome completo, il datore di lavoro e le risposte alle domande di sicurezza dell'utente a un server attaccante, senza alcuna indicazione visibile. L'attacco sfrutta un buco nel design del tool web_fetch di Claude, progettato per evitare esfiltrazione: un prompt injection nella pagina web letta può persuadere l'agente a codificare dati privati nei parametri della URL di fetch. Simon Willison segnala che il vettore d'attacco è la "lethal trifecta": accesso a dati privati (memoria delle interazioni passate), tool per leggere contenuti online, e tool per fare richieste web in uscita.
**Fonti:** [Simon Willison](https://simonwillison.net/2026/Jul/15/claude-web-fetch-exfiltration/#atom-everything) — analisi tecnica del bug nel design di web_fetch e del vettore lethal trifecta; [Lobste.rs](https://ayush.digital/blog/the-memory-heist) — descrizione completa dell'attacco e proof-of-concept.

### 5. AsyncAPI npm compromesso: supply chain attack con Trojan RAT infostealer nei pacchetti @asyncapi
Cinque versioni malevole di pacchetti AsyncAPI sono state pubblicate su npm in un attacco alla supply chain che ha distribuito un RAT con capacità info-stealing. L'attaccante ha sfruttato una GitHub Actions workflow mal configurata e pushato pacchetti trojanizzati nel namespace @asyncapi, con un totale cumulativo di oltre 2,25 milioni di download settimanali. Il 14 luglio due repository GitHub AsyncAPI sono stati compromessi. Secondo OX Security e SafeDep, i pacchetti distribuiscono un multi-stage botnet loader. I pacchetti coinvolti: @asyncapi/generator-helpers@1.1.1, @asyncapi/generator-components@0.7.1, @asyncapi/generator@3.3.1, @asyncapi/specs (v6.11.2 e v6.11.2-alpha.1).
**Fonti:** [BleepingComputer](https://www.bleepingcomputer.com/news/security/-asyncapi-npm-packages-infected-with-credential-stealing-malware/) — cinque pacchetti malevoli, GitHub Actions compromesso, 2,25M download; [The Hacker News](https://thehackernews.com/2026/07/compromised-asyncapi-npm-packages.html) — dettagli tecnici: multi-stage botnet loader, nomi versioni e vendor di sicurezza coinvolti.

### 7. Cursor non patchato: un repository malevolo esegue codice automaticamente su Windows, 7M utenti a rischio
Una vulnerabilità non patchata in Cursor su Windows permette l'esecuzione di codice quando un sviluppatore apre un repository nell'applicazione. Il difetto è diretto: se un file git.exe si trova nella root del progetto, Cursor lo esegue automaticamente senza avvisare l'utente o chiedere approvazione. Il binary gira con i privilegi dell'utente, con accesso a chiavi SSH e token cloud. Cursor continua a rieseguirlo finché il progetto resta aperto. Nessun prompt injection o agente AI coinvolto: l'apertura del folder è l'intero exploit. Cursor ha oltre 7 milioni di utenti attivi.
**Fonti:** [SecurityWeek](https://www.securityweek.com/unpatched-cursor-vulnerability-exposes-users-to-code-execution/) — descrizione tecnica del flaw, 7M utenti, esecuzione automatica; [The Hacker News](https://thehackernews.com/2026/07/cursor-flaw-lets-malicious-cloned.html) — dettagli: nessuna interazione richiesta, binary si riesegue; [Dark Reading](https://www.darkreading.com/application-security/2-click-cursor-exploit-dev-environment-takeover) — 2-click exploit, takeover ambiente dev.
